# AgaLife — цифровая площадка для жителей Агинского Бурятского округа

## Миссия, позиционирование и слоган

**Слоган:** «Там, где живёт округ»

**Одна фраза о проекте:** Цифровой дом Агинского Бурятского округа.

**Почему мы это делаем:** жители Агинского Бурятского округа заслуживают современный цифровой сервис, который объединяет культурную жизнь, мероприятия и людей. Округ живёт благодаря своим жителям, традициям и событиям — наша задача сделать так, чтобы эту жизнь было легко увидеть.

**Ключевой сценарий:** в округ приезжает гость и не знает, куда пойти и что происходит. Ему говорят: «Зайди на AgaLife» — и с одного взгляда становится понятно, чем округ живёт прямо сейчас.

**Как это реализуется в MVP:** самый прямой способ показать жизнь округа — афиша мероприятий. Житель заходит и сразу видит, что происходит рядом с ним.

## 1. О проекте

- **Формат старта:** веб-сайт (адаптивный, удобно и на компьютере, и на телефоне)
- **В перспективе:** мобильное приложение
- **Аудитория:** жители округа — Агинское, Могойтуй, Дульдурга и другие сёла округа
- **Фокус MVP:** афиша мероприятий — лицо и центральный раздел площадки, обеспечивает доверие и посещаемость
- **Модель развития:** рядом с афишей развивается барахолка — платный раздел объявлений, который зарабатывает деньги, не будучи лицом проекта

## 2. Функционал MVP

### 2.1 Афиша мероприятий (лицо проекта, бесплатно)

- Список / календарь всех мероприятий округа: концерты, спорт, ярмарки, сходы жителей, культурные и национальные праздники (Сагаалган, Сурхарбан и т.д. — карточки событий, без отдельного раздела про традиции)
- Фильтры: **по селу/посёлку** («рядом со мной»), **по времени** (сегодня / на этой неделе / позже), по категории события
- Карточка события: название, дата и время, место (с картой), описание, организатор, контакты
- Публикация событий — через простую админ-панель (модератор добавляет событие), в дальнейшем платный личный кабинет у организаторов мероприятий

### 2.2 Барахолка округа (платный движок под капотом)

- Специализированная доска объявлений с упором на категории, где у продавца есть реальный бюджет на сделку: **авто и сельхозтехника, скот и сельхозпродукция, недвижимость, стройматериалы** (плюс общая категория «разное»)
- Обычное объявление — **бесплатно**: это магнит трафика и способ быстро набрать критическую массу объявлений
- Платные функции:
  - Поднятие / топ объявления на фиксированный срок
  - Бейдж «Срочная продажа»
  - Профиль постоянного продавца (автосалон, риелтор, фермерское хозяйство) — несколько активных объявлений без ограничений, расширенная карточка
- Раздел не выносится на главную страницу — доступен из меню, чтобы не конкурировать с афишей за внимание и не размывать «лицо» проекта
- **Дальше:** раздел «Работа» — в т.ч. сезонная (сенокос, уборка урожая) — платное размещение вакансий для работодателей, тоже под капотом, не на главной

## 3. Модель монетизации

- **Афиша** — бесплатная, это лицо проекта: обеспечивает доверие, ежедневные заходы и узнаваемость бренда «Там, где живёт округ»
- **Барахолка** — платный движок: деньги приносят продавцы коммерчески значимых категорий (авто, недвижимость, сельхозтехника, скот), которым есть прямой смысл платить за видимость при сделке на десятки–сотни тысяч рублей
- На афише — никакой рекламы и баннеров, монетизация полностью вынесена в барахолку
- Следующий платный блок после барахолки — раздел «Работа» (платные вакансии для работодателей)

## 4. Структура разделов сайта

```
Главная = Афиша (список/календарь + фильтры по селу и дате)
└── Карточка события

Барахолка (платный раздел, доступен из меню, не с главной)
├── Категории: авто/сельхозтехника, недвижимость, скот/сельхозпродукция, стройматериалы, разное
├── Карточка объявления (бесплатная или с платным поднятием/бейджем)
└── Профиль продавца (обычный или платный «постоянный продавец»)
```

Новости, форум и отдельный раздел «Культура» — по-прежнему вне MVP.

## 5. Главная страница = Афиша

- **Хиро-блок (минимальный)**: слоган «Там, где живёт округ» + одна строка о сути проекта
- **Афиша сразу под хиро-блоком** — основное содержимое главной: список ближайших событий с фильтрами по селу и дате
- **Фильтр «рядом со мной»** — по умолчанию показывает события в селе жителя (если указано) или во всём округе
- Барахолка на главную не выносится вообще — только ссылка в меню

## 6. Технологический стек (рекомендация)

| Часть | Технология | Почему |
|---|---|---|
| Фронтенд | **Next.js** (React) + **Tailwind CSS** | Компонентный подход, удобно наращивать функциональность постепенно |
| База данных, авторизация, хранение фото | **Supabase** (Postgres) | Готовая авторизация, база данных и хранилище файлов «из коробки», есть бесплатный тариф |
| Приём платежей (барахолка) | **ЮKassa** или **CloudPayments** | Стандарт для приёма платежей от физлиц в РФ, простая интеграция |
| Карты | **Яндекс Карты API** или **2ГИС API** | Лучше покрытие для Забайкальского края, чем у Google Maps |
| Хостинг | **Vercel** (сайт) + **Supabase Cloud** (база данных) | Бесплатных тарифов достаточно для MVP |
| Мобильное приложение (позже) | **React Native / Expo** | Переиспользует логику и API от веб-версии |

## 7. Модель данных (черновик)

- **events**: id, название, описание, дата/время, место, село, категория, организатор, контакты
- **classifieds**: id, заголовок, описание, цена, категория (авто/сельхозтехника, недвижимость, скот/сельхозпродукция, стройматериалы, разное), контакты, автор, статус модерации, **тип размещения** (обычное / топ / срочная продажа), **срок действия платного размещения**
- **seller_profiles**: id, тип (обычный / платный «постоянный продавец»), название, категория, контакты, статус подписки, дата окончания подписки
- **payments**: id, user_id, тип услуги (топ объявления / бейдж / подписка продавца), сумма, дата, статус
- **users**: id, имя, село, роль

## 8. Этапы разработки

1. **Каркас + Афиша** — лицо проекта: главная страница = список/календарь событий, карточка события
2. **Фильтры афиши** — по селу и по дате («рядом со мной», «сегодня», «на этой неделе»)
3. **Барахолка, базовая версия** — бесплатные объявления, категории, модерация
4. **Приём платежей в барахолке** — платное поднятие, бейджи, платные профили продавцов
5. **Деплой и обратная связь от жителей**
6. **Дальше** — раздел «Работа»; вопрос новостей/форума/культуры решаем по фидбеку от жителей
